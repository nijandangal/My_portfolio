Project 1: Website Controlled Car (ESP32 Web Server + Web Client)

Build your ESP32 car with L298N motor driver, then control it from a web page. Method 1 runs ESP32 as webserver and method 2 uses your website as webclient sending commands to car endpoints.

ESP32 Robot Car

Two-wheel car control with motor PWM and direction mapping.

Webserver Mode

ESP32 hosts control dashboard and accepts /F /B /L /R /S /speed commands.

Webclient Mode

Website sends HTTP commands to ESP32 for remote control from your portfolio page.

Failsafe

Automatic stop if no command received for 5 seconds.

Hardware & Software Requirements

ESP32 Dev Board

ESP32 DevKit or similar (must support LEDC PWM pins).

L298N Motor Driver

Dual H-bridge driver for two DC motors, with ENA/ENB PWM inputs.

Motor Power

7.4V–12V battery pack for motors; common ground with ESP32.

WiFi Network

2.4GHz router. ESP32 must be on same network as your browser for control.

Arduino IDE

Install Arduino IDE 2.x and ESP32 board support (https://dl.espressif.com/dl/package_esp32_index.json).

Method 1: Web Server Code (ESP32 hosts dashboard)

This firmware listens on port 80 and accepts requests like /F, /B, /L, /R, /S, and /speed=200. It also shows a responsive HTML dashboard.

#include <WiFi.h>
const char* ssid = "YOUR_SSID";
const char* password = "YOUR_PASSWORD";
WiFiServer server(80);
#define ENA 25
#define IN1 16
#define IN2 17
#define IN3 18
#define IN4 19
#define ENB 26
const int speedThreshold = 160;
int motorSpeed = 200;
String currentStatus = "Stopped";
unsigned long lastCommandTime = 0;
void setup(){ Serial.begin(115200); pinMode(IN1,OUTPUT);pinMode(IN2,OUTPUT);pinMode(IN3,OUTPUT);pinMode(IN4,OUTPUT); ledcSetup(0,20000,8); ledcSetup(1,20000,8); ledcAttachPin(ENA,0); ledcAttachPin(ENB,1); stopCar(); WiFi.begin(ssid,password); while(WiFi.status()!=WL_CONNECTED){delay(100);} server.begin(); }
void applyPwm(){ if(motorSpeed>=speedThreshold){ ledcWrite(0,motorSpeed); ledcWrite(1,motorSpeed);} else { ledcWrite(0,0); ledcWrite(1,0);} }
void setSpeed(int s){ motorSpeed = constrain(s,speedThreshold,242); applyPwm(); currentStatus="Speed "+String(motorSpeed); lastCommandTime=millis(); }
void stopCar(){ digitalWrite(IN1,LOW);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,LOW); ledcWrite(0,0); ledcWrite(1,0); currentStatus="Stopped";}
void forward(){ applyPwm(); digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); currentStatus="Forward"; lastCommandTime=millis(); }
void backward(){ applyPwm(); digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); currentStatus="Backward"; lastCommandTime=millis(); }
void left(){ applyPwm(); digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); currentStatus="Left"; lastCommandTime=millis(); }
void right(){ applyPwm(); digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); currentStatus="Right"; lastCommandTime=millis(); }
void handleClient(){ WiFiClient client = server.available(); if(!client)return; String req=client.readStringUntil('\r'); client.flush(); if(req.indexOf("/F")!=-1) forward(); else if(req.indexOf("/B")!=-1) backward(); else if(req.indexOf("/L")!=-1) left(); else if(req.indexOf("/R")!=-1) right(); else if(req.indexOf("/S")!=-1) stopCar(); else if(req.indexOf("/speed=")!=-1){ int v=req.substring(req.indexOf("/speed=")+7).toInt(); setSpeed(v);} client.print("HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n"); client.print("
ESP32 Car - "+currentStatus+"
"); client.stop(); }
void loop(){ handleClient(); if(millis()-lastCommandTime>5000) stopCar(); }

Method 2: Web Client from Website to Control Motors

Use your website JavaScript to send direct HTTP commands to ESP32's webserver endpoints. This is the exact method to control motors from your portfolio app.

<div class="car-client">
  <input id="carIp" placeholder="ESP32 IP"/>
  <button onclick="connectCar()">Connect</button>
  <button onclick="cmd('F')">Forward</button>
  <button onclick="cmd('B')">Backward</button>
  <button onclick="cmd('L')">Left</button>
  <button onclick="cmd('R')">Right</button>
  <button onclick="cmd('S')">Stop</button>
  <input type="range" id="speedSlider" min="160" max="242" value="200" oninput="setSpeed(this.value)"/>
</div>
let baseUrl='';
function connectCar(){ baseUrl='http://'+document.getElementById('carIp').value; }
function cmd(action){ if(!baseUrl){alert('Set IP'); return;} fetch(baseUrl+'/'+action); }
function setSpeed(v){ if(!baseUrl){return;} fetch(baseUrl+'/speed='+v); }{ if(baseUrl)fetch(baseUrl+'/status'); },3000);

Use this method with your own HTML page to avoid duplication. Keep one mode active: either web server control or web client control. This sample uses only webclient commands to the same ESP32 endpoint.

Dos & Don'ts

Do

  • Use stable power for motors and ESP32.
  • Keep command timeout to stop if no signal received.
  • Verify IP from Serial Monitor before hitting controls.

Don't

  • Don't run motors at 255 full speed without testing.
  • Don't forget common ground between batteries and ESP32.
  • Don't use 5GHz WiFi for ESP32 connection.

Final Check

After uploading, open Serial Monitor at 115200. Confirm IP. Then open either the ESP32 dashboard URL for server mode or your website control panel for webclient mode. Test each direction and speed slider.

Project 2: Bluetooth Controlled Car

Single-mode manual car control with your phone app via Bluetooth. Based on file Esp32_Bluetooth_L298N_robot_car_MIT_APP_VoiceControlled.ino.

Key Details

1) Hardware

  • ESP32 Dev Board (GPIO 16,17,18,19,25,26 used)
  • L298N Motor Driver + Dual DC Motors
  • Bluetooth module (HC-05/HC-06 or built-in ESP32 BTSerial)
  • Battery (7.4V Li-ion or 6V pack) + common GND

2) Software

  • Arduino IDE 2.x
  • BluetoothSerial library (built-in ESP32)
  • Set board: ESP32 Dev Module, COM port, upload speed 115200
  • File path: Robo-Cars/Esp32_Bluetooth_L298N_robot_car_MIT_APP_VoiceControlled/Esp32_Bluetooth_L298N_robot_car_MIT_APP_VoiceControlled.ino

3) Steps

  1. Wire motors to L298N and ESP32 pins IN1/IN2/IN3/IN4/ENA/ENB per code defines.
  2. Connect Bluetooth RX/TX (HC-05 TX → ESP32 RX, HC-05 RX → ESP32 TX, 5V power).
  3. Upload code and pair phone app to ESP32_RobotCar.
  4. Send single-character commands: F/B/L/R/S and speed with #180.

4) Watchouts & Guidelines

  • Use proper power (>=2A) for motors; unstable power causes resets.
  • Set motorSpeed within 100–230 to avoid overcurrent and jitter.
  • Implement failsafe: if no command in 2s, the car stops automatically.
  • Check direction and wiring before running; wrong wiring may reverse controls.

Project 3: Obstacle Avoidance Car

Single-mode obstacle avoidance: choose IR or Ultrasonic method as per your sensor set. Includes Bluetooth manual fallback in the IR file. Uses UNO/ESP32 architecture from repository.

Method A: IR Obstacle Avoidance (with Bluetooth manual toggle)

File: UNO_Multimode_Bluetooth_HC-06_and_Obstacle_3_IR__L298P_FUNDO/UNO_Multimode_Bluetooth_HC-06_and_Obstacle_3_IR__L298P_FUNDO.ino

Hardware

  • Arduino UNO (or ESP32 with similar pin update)
  • 3 IR sensors (left, center, right)
  • L298P driver + dual DC motors
  • HC-06 Bluetooth module (for manual mode input)
  • Optional buzzer for beep signals and mode feedback

How It Works

  • Manual mode reads BT commands F/B/L/R/S and sets motor outputs.
  • Obstacle mode reads IR sensors and chooses forward/back/turn actions by simple rules.
  • Mode toggles with switch (momentary button), single mode at a time.

Method B: Ultrasonic Obstacle Avoidance

File: UNO_Multimode_Bluetooth_HC-06_Obstacle_3_Ultrasonic__L298P/ (use ultrasonic HC-SR04, update pins as needed). This method checks distance and stops/reverses at threshold.

Hardware

  • HC-SR04 ultrasonic sensor (Trig/Echo)
  • Dual motor driver and motors
  • Arduino UNO or ESP32 (with 5V logic compatibility)

Key Tips

  • Keep sensor at a stable mounting height and angle for reliable detection.
  • Use a 10µF capacitor near motor supply to reduce noise spikes.
  • Tune thresholds carefully (e.g., 10–15 cm for safe emergency stop).

Software + Steps

  1. Open the recommended .ino file in Arduino IDE.
  2. Set correct pins for motors and sensors according to your board.
  3. Update Bluetooth name and baud if needed (HC-06 default 9600).
  4. Upload and verify serial output for mode and sensor readings.
  5. Test in a clear area and gradually add obstacles for tuning.

Guidelines & Things to Watch

  • Always use common ground between logic and driver power supplies.
  • Do not run motors at full speed without checking load and battery capacity.
  • Debounce mode switch input and use status LEDs for mode feedback.
  • Check sensor values in Serial Monitor before final deployment.
  • For IR, avoid direct sunlight and reflective floor surfaces that can misread values.
