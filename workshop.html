Project 1.1: WiFi Controlled Car - ESP32 WebServer

Build an ESP32 robot car that hosts its own control dashboard. The ESP32 runs a web server on port 80, accepts movement commands and speed control, with automatic failsafe.

ESP32 Robot Car

Two-wheel car control with L298N motor driver.

Web Server Mode

Hosts dashboard and accepts /F /B /L /R /S /speed commands.

Local Network

Control from any browser on the same WiFi network.

Failsafe

Auto-stop after 5 seconds without command.

Hardware Requirements

ESP32 Dev Board

ESP32 DevKit with sufficient GPIO and PWM pins.

L298N Motor Driver

Dual H-bridge driver for two DC motors with PWM input.

DC Motors

Two motors (3–12V) for wheel control.

Power

7.4V–12V battery pack; common ground with ESP32.

WiFi Network

2.4GHz router for ESP32 connection.

Arduino IDE

Arduino IDE 2.x with ESP32 board support.

Pin Mapping

ComponentESP32 GPIOFunction
ENA (Motor A PWM)25Speed control Motor A
IN1 / IN216 / 17Motor A direction
ENB (Motor B PWM)26Speed control Motor B
IN3 / IN418 / 19Motor B direction
Motor driver VCCBattery 7–12VMotor power
Motor driver GNDGNDCommon ground

Web Server Code

ESP32 listens on port 80 and accepts requests like /F, /B, /L, /R, /S, and /speed=200.

#include <WiFi.h>
const char* ssid = "YOUR_SSID";
const char* password = "YOUR_PASSWORD";
WiFiServer server(80);
#define ENA 25
#define IN1 16
#define IN2 17
#define IN3 18
#define IN4 19
#define ENB 26
const int speedThreshold = 160;
int motorSpeed = 200;
String currentStatus = "Stopped";
unsigned long lastCommandTime = 0;
void setup(){ Serial.begin(115200); pinMode(IN1,OUTPUT);pinMode(IN2,OUTPUT);pinMode(IN3,OUTPUT);pinMode(IN4,OUTPUT); ledcSetup(0,20000,8); ledcSetup(1,20000,8); ledcAttachPin(ENA,0); ledcAttachPin(ENB,1); stopCar(); WiFi.begin(ssid,password); while(WiFi.status()!=WL_CONNECTED){delay(100);} server.begin(); }
void applyPwm(){ if(motorSpeed>=speedThreshold){ ledcWrite(0,motorSpeed); ledcWrite(1,motorSpeed);} else { ledcWrite(0,0); ledcWrite(1,0);} }
void setSpeed(int s){ motorSpeed = constrain(s,speedThreshold,242); applyPwm(); currentStatus="Speed "+String(motorSpeed); lastCommandTime=millis(); }
void stopCar(){ digitalWrite(IN1,LOW);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,LOW); ledcWrite(0,0); ledcWrite(1,0); currentStatus="Stopped";}
void forward(){ applyPwm(); digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); currentStatus="Forward"; lastCommandTime=millis(); }
void backward(){ applyPwm(); digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); currentStatus="Backward"; lastCommandTime=millis(); }
void left(){ applyPwm(); digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); currentStatus="Left"; lastCommandTime=millis(); }
void right(){ applyPwm(); digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); currentStatus="Right"; lastCommandTime=millis(); }
void handleClient(){ WiFiClient client = server.available(); if(!client)return; String req=client.readStringUntil('\r'); client.flush(); if(req.indexOf("/F")!=-1) forward(); else if(req.indexOf("/B")!=-1) backward(); else if(req.indexOf("/L")!=-1) left(); else if(req.indexOf("/R")!=-1) right(); else if(req.indexOf("/S")!=-1) stopCar(); else if(req.indexOf("/speed=")!=-1){ int v=req.substring(req.indexOf("/speed=")+7).toInt(); setSpeed(v);} client.print("HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n"); client.print("
ESP32 Car - "+currentStatus+"
"); client.stop(); }
void loop(){ handleClient(); if(millis()-lastCommandTime>5000) stopCar(); }

Setup & Testing

  1. Wire L298N pins according to the pin mapping table above.
  2. Update WiFi credentials (SSID/password) in the code.
  3. Upload to ESP32 using Arduino IDE.
  4. Open Serial Monitor at 115200 baud to see ESP32's IP address.
  5. Open browser and navigate to your ESP32 IP (e.g., http://192.168.1.100).
  6. Test each direction button and speed slider.

Tips

Do

  • Use stable power for motors.
  • Keep command timeout mechanism to stop if no signal.
  • Verify IP in Serial Monitor before testing.

Don't

  • Don't use 5GHz WiFi for ESP32.
  • Don't skip common ground between battery and ESP32.
  • Don't run motors at full 255 PWM without testing.

let baseUrl = '';
function connectCar() {
  const ip = document.getElementById('carIp').value.trim();
  if (!ip) return alert('Enter ESP32 IP');
  baseUrl = 'http://' + ip;
  fetch(baseUrl + '/status')
    .then(r => r.text())
    .then(d => alert('Connected'))
    .catch(() => alert('Connection failed. Check IP.'));
}
function cmd(action) {
  if (!baseUrl) return alert('Connect to car first');
  fetch(`${baseUrl}/${action}`).catch(() => alert('Command failed'));
}
function setSpeed(v) {
  if (!baseUrl) return;
  fetch(`${baseUrl}/speed=${v}`).catch(() => {});
}

Integration Steps

  1. Add HTML control panel to your portfolio page.
  2. Include JavaScript functions in your script.js.
  3. Ensure ESP32 runs HTTP server firmware (Project 1.1).
  4. Find ESP32's IP from Serial Monitor.
  5. Enter IP in control panel and click Connect.
  6. Use buttons to control your car remotely.

Project 2.1: Bluetooth Controlled Car - ESP32 + L298N

Control your robot car via Bluetooth using a mobile app. Perfect for hand-held control without WiFi dependency.

Bluetooth Classic

Built-in ESP32 Bluetooth support.

Mobile Control

Send F/B/L/R/S and #speed commands from app.

Speed Control

Real-time PWM adjustment (100–230).

Auto-Stop

Stops after 2–5 seconds without command.

Hardware & Components

ESP32

Built-in Bluetooth classic support.

L298N

Dual H-bridge motor driver.

DC Motors

Two motors (3–12V) with wheels.

Power

7.4V–12V battery pack.

BT App

Android: Serial Bluetooth Terminal app.

Pin Configuration

FunctionESP32 PinPurpose
ENA (Motor A)25PWM speed control
IN1 / IN216 / 17Motor direction
ENB (Motor B)26PWM speed control
IN3 / IN418 / 19Motor direction
GNDGNDCommon ground

Firmware Code

#include "BluetoothSerial.h"
BluetoothSerial SerialBT;
#define ENA 25, IN1 16, IN2 17, IN3 18, IN4 19, ENB 26
int motorSpeed=180, minSpeed=100, maxSpeed=230;
unsigned long lastCommandTime = 0;
void setup(){ 
  Serial.begin(115200); 
  SerialBT.begin("ESP32_RobotCar"); 
  pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT); pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT); 
  ledcAttachPin(ENA,0); ledcAttachPin(ENB,1); 
  ledcSetup(0,5000,8); ledcSetup(1,5000,8); 
  stopCar(); 
}
void loop(){ 
  if(SerialBT.available()){ 
    char cmd = SerialBT.read(); 
    if(cmd=='#'){ int val=SerialBT.parseInt(); motorSpeed = constrain(val,minSpeed,maxSpeed); }
    else { switch(cmd){
      case 'F': case 'B': case 'L': case 'R': case 'S': handleCommand(cmd); break;
    }} 
    lastCommandTime = millis();
  } 
  if(millis()-lastCommandTime > 2000) stopCar();
}
Send via Bluetooth App:
F  -> Forward
B  -> Backward
L  -> Left Turn
R  -> Right Turn
S  -> Stop
#100 -> Set speed 100
#180 -> Set speed 180
#230 -> Set speed 230

Use Serial Bluetooth Terminal or MIT App Inventor.

Setup Instructions

  1. Install Serial Bluetooth Terminal on Android.
  2. Upload code to ESP32 with correct board & COM port.
  3. Power ESP32 and open Bluetooth settings.
  4. Pair with "ESP32_RobotCar".
  5. Open Serial Bluetooth Terminal and connect.
  6. Send F/B/L/R/S for direction control.
  7. Send #XXX to adjust speed.

MM.1: Advanced Multimode Car - WiFi + Bluetooth + OLED

Single firmware with multiple control modes: WiFi webserver, Bluetooth app, and OLED status display. Switch modes seamlessly.

WiFi Mode

Host control dashboard on ESP32.

Bluetooth Mode

Control via mobile app simultaneously.

OLED Display

Real-time status on 128x64 screen.

Mode Switch

Toggle between modes via GPIO pin.

Hardware Requirements

ESP32

With WiFi & Bluetooth.

L298N

Motor driver.

SSD1306 OLED

128x64 I2C display.

Motors & Battery

Two 3–12V motors, 7.4V–12V power.

WiFi Network

2.4GHz router.

BT Terminal App

Serial Bluetooth Terminal.

Pin Configuration

ComponentESP32 PinFunction
ENA / ENB25 / 26Motor PWM
IN1–IN416,17,18,19Motor direction
OLED SDA / SCL21 / 22I2C display
Mode Pin33Mode toggle

Key Features

  • Single firmware with dual control modes
  • WiFi webserver & Bluetooth simultaneously
  • OLED real-time status display
  • GPIO-based mode switching
  • Automatic failsafe (5 sec timeout)
  • Speed control 160-242 PWM

Integration Steps

  1. Install: BluetoothSerial, U8g2 libraries.
  2. Update WiFi credentials in code.
  3. Upload to ESP32.
  4. Test Bluetooth mode first.
  5. Toggle GPIO 33 to GND for WiFi mode.
  6. Both modes work simultaneously.
